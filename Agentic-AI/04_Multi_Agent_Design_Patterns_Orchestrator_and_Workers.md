# Multi-Agent Design Patterns: Orchestrator and Workers

When building solutions for complex, multi-layered industrial tasks, relying on a single monolithic agent frequently causes systemic failures. As an agent's prompt instructions, tool options, and context history expand, its attention vectors suffer from dilution, leading to severe reasoning drops and misrouted function calls. To scale engineering limits, architectures pivot to **Multi-Agent Design Patterns**, segregating workloads among a cluster of narrow, specialized worker components coordinated by a central supervisor logic.

This document breaks down task decomposition math, multi-agent communication protocols, and a zero-dependency orchestrator-worker framework built from scratch.

---

## 1. The Orchestrator-Worker (Supervisor) Topology

The Orchestrator-Worker design pattern mirrors traditional human engineering teams: a highly capable supervisor breaks down an unstructured objective into discrete sub-tasks, assigns those sub-tasks to distinct specialized agents, and aggregates their independent outputs into a singular synthesized conclusion.

* **The Orchestrator (Supervisor):** Acts as the central route dispatcher. It holds the high-level roadmap execution state but does not do raw tactical work. It determines when a sub-task is complete or if it needs to be sent back to a worker for remediation.
* **The Workers:** Specialized agent scopes stripped of unnecessary tools and background noise. One worker might specialize purely in executing SQL commands, another in auditing string vulnerabilities, and another in text summarization. This strict boundary optimizes their attention density.

---

## 2. Mathematical Formalization of Task Decomposition

Let $G$ represent the global user objective statement. The Orchestrator applies a decomposition mapping function $\mathcal{D}$ to break $G$ down into a sequence of $N$ dependent, ordered sub-tasks:

$$\mathcal{D}(G) = \{t_1, t_2, \dots, t_N\}$$

Each sub-task $t_i$ is explicitly routed to a targeted worker agent $W_k$ selected from an available agent swarm fleet $\mathcal{W} = \{W_1, W_2, \dots, W_m\}$. The execution result $r_i$ generated by worker $W_k$ is formalized as a function of the task context and the worker's internal capability weights:

$$r_i = W_k(t_i \mid r_{<i})$$

Where $r_{<i}$ represents the aggregated structural results compiled from all prior sub-task iterations. The final system payload delivery $Y$ is synthesized by the supervisor combining the global historical trace profile:

$$Y = \text{Orchestrator}(G, r_1, r_2, \dots, r_N)$$

## 3. Production-Grade Implementation: Multi-Agent Coordination Grid

Below is a complete, self-contained Python multi-agent system. It establishes a supervisor router that interprets a programming query, splits the workload, sequentially invokes a specialized **CodeGenerator Worker** and a **SecurityAuditor Worker**, and returns a consolidated output string.

```python
import json
from typing import List, Dict, Any

class WorkerAgent:
    """Specialized worker unit with restricted behavioral roles."""
    def __init__(self, name: str, specialization_prompt: str):
        self.name = name
        self.specialization_prompt = specialization_prompt

    def execute_task(self, sub_task_input: str) -> str:
        """Simulates targeted localized execution output."""
        if self.name == "CodeGenerator":
            return (
                "```python\n"
                "def process_user_login(user_input):\n"
                "    # Dynamic query assembly string\n"
                "    query = f'SELECT * FROM users WHERE input = {user_input}'\n"
                "    return db.execute(query)\n"
                "```"
            )
        elif self.name == "SecurityAuditor":
            return (
                "CRITICAL AUDIT VULNERABILITY ALERT:\n"
                " - Detected SQL Injection vulnerability via string interpolation formatting on line 3.\n"
                " - Remediation Strategy: Enforce explicit parameterized execution bindings immediately."
            )
        return "Task processed successfully by quiet worker."


class MultiAgentOrchestrator:
    """Central supervisor coordinating task distribution and final delivery synthesis."""
    def __init__(self):
        self.workers: Dict[str, WorkerAgent] = {}
        self._initialize_swarm()

    def _initialize_swarm(self):
        """Provision individual specialized agents into the network registry."""
        self.workers["codegen"] = WorkerAgent(
            name="CodeGenerator",
            specialization_prompt="You are an expert backend engineer emitting clean, localized Python blocks."
        )
        self.workers["sec_audit"] = WorkerAgent(
            name="SecurityAuditor",
            specialization_prompt="You are an elite static analysis security inspector tracing string injections."
        )

    def route_and_synthesize(self, global_goal: str):
        print(f"=== [Supervisor] Initialized Goal: '{global_goal}' ===")
        
        # Step 1: Orchestrator decomposes problem space into discrete assignments
        print("\n1. [Orchestrator Node] Mapping task dependencies...")
        sub_tasks = [
            {"worker": "codegen", "instruction": "Generate Python authentication lookup function."},
            {"worker": "sec_audit", "instruction": "Verify raw source text against cryptographic vulnerabilities."}
        ]
        
        execution_results = []
        
        # Step 2: Sequential dispatch loop execution across worker registries
        for index, task in enumerate(sub_tasks):
            target_worker = self.workers[task["worker"]]
            print(f" >> Dispatching Sub-Task #{index + 1} to [{target_worker.name}]...")
            
            # Combine historical context tracking into worker space input
            worker_output = target_worker.execute_task(task["instruction"])
            execution_results.append(worker_output)
            
            print(f" << [{target_worker.name}] Completed operational routine.")
            
        # Step 3: Central Supervisor consolidates individual blocks into final report
        print("\n2. [Orchestrator Node] Compiling and synthesizing final structural summary...")
        final_delivery_payload = (
            "=== SYSTEM ARCHITECTURE REVIEW REPORT ===\n\n"
            "--- PRODUCED COMPONENT RAW CODE ---\n"
            f"{execution_results[0]}\n\n"
            "--- COMPLIANCE SECURITY AUDIT TRACE ---\n"
            f"{execution_results[1]}\n\n"
            "Status: ACTION REQUIRED. Reject deployment until parameterization bindings are corrected."
        )
        
        return final_delivery_payload


# --- Verification Execution Sandbox ---
if __name__ == "__main__":
    orchestrator_system = MultiAgentOrchestrator()
    prompt_target = "Build a fast user data retrieval authentication pipeline for database integrations."
    
    final_output_report = orchestrator_system.route_and_synthesize(prompt_target)
    print("\n" + final_output_report)
```
