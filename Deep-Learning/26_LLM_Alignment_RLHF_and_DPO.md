# 26. LLM Alignment: RLHF and Direct Preference Optimization (DPO)

Once an autoregressive Large Language Model finishes pre-training and supervised fine-tuning (SFT), it excels at token prediction but lacks alignment with human intent. It may generate toxic, deceptive, or unhelpful responses. **LLM Alignment** shifts the model from predicting the next token to optimizing for human values, safety, and helpfulness.

---

## 1. Reinforcement Learning from Human Feedback (RLHF)

The classic alignment paradigm popularized by InstructGPT and ChatGPT splits the training pipeline into three sequential stages:
### The Bradley-Terry Preference Model
Human evaluators choose between two model responses given a prompt $x$: a winning response $y_w$ and a losing response $y_l$. The probability that a human prefers $y_w$ over $y_l$ is modeled using a scalar reward function $r_\psi(x, y)$:

$$P(y_w \succ y_l \mid x) = \sigma(r_\psi(x, y_w) - r_\psi(x, y_l))$$

Where $\sigma(z) = \frac{1}{1 + e^{-z}}$ is the standard sigmoid function.

### The Proximal Policy Optimization (PPO) Objective
In Stage 3, the policy network (the LLM) $\pi_\theta$ is updated to maximize the scalar reward while applying a Kullback-Leibler (KL) penalty to prevent it from drifting too far from the initial unaligned base SFT model $\pi_{\text{SFT}}$:

$$\mathcal{L}_{\text{PPO}}(\theta) = \mathbb{E}_{(x, y) \sim \mathcal{D}} \left[ r_\psi(x, y) \right] - \beta \, D_{\text{KL}}(\pi_\theta(y \mid x) \parallel \pi_{\text{SFT}}(y \mid x))$$

**The PPO Bottleneck:** This loop requires maintaining up to **four massive models simultaneously in VRAM**: the active policy ($\pi_\theta$), the frozen reference network ($\pi_{\text{SFT}}$), the scalar reward estimator ($r_\psi$), and the critic/value network. This makes PPO computationally heavy and notoriously unstable to hyperparameter tuning.

---

## 2. Direct Preference Optimization (DPO)

Introduced by Rafailov et al. (2023), **Direct Preference Optimization** eliminates the need for an explicit reward model or an active RL loop entirely. 

By applying a mathematical change of variables to the Bradley-Terry preference framework, DPO proves that the implicit reward function can be expressed directly as the log-ratio of the active policy to the reference model. This allows the model to be fine-tuned directly on preference pairs using a clean, standard binary cross-entropy loss.

### The Analytical DPO Loss Function:
$$\mathcal{L}_{\text{DPO}}(\theta; \pi_{\text{SFT}}) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{\text{SFT}}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{\text{SFT}}(y_l \mid x)} \right) \right]$$

Where:
* $\pi_\theta(y \mid x)$: The sequence log-probabilities generated by the active model being trained.
* $\pi_{\text{SFT}}(y \mid x)$: The sequence log-probabilities from the frozen, pre-trained base model.
* $\beta$: A hyperparameter controlling the strength of the implicit KL penalty constraint. It keeps the distribution grounded.

---

## 3. Structural Comparison

| Optimization Property | Standard RLHF (PPO) | Direct Preference Optimization (DPO) |
| :--- | :--- | :--- |
| **Active Models in VRAM** | 4 (Policy, Reference, Reward, Critic) | 2 (Active Policy, Frozen Reference) |
| **Mathematical Approach** | Reinforcement Learning Actor-Critic | Supervised Classification (BCE) |
| **Training Stability** | Low (Sensitive to values, reward hacking) | High (Stable gradient optimization) |
| **Compute Overhead** | High (Requires slow online sampling loops) | Low (Token logs calculated offline/batched) |

---

## 4. Implementation in Python (PyTorch)

This script demonstrates how to implement a standalone **DPO Loss Function** from scratch in PyTorch. It processes raw token logits from both an active policy and a reference model across winning and losing sequences.

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class DirectPreferenceOptimizationLoss(nn.Module):
    """
    Implements the implicit mathematical loss objective for Direct Preference Optimization (DPO).
    """
    def __init__(self, beta: float = 0.1):
        super(DirectPreferenceOptimizationLoss, self).__init__()
        self.beta = beta

    def get_batch_logps(self, logits: torch.Tensor, labels: torch.Tensor) -> torch.Tensor:
        """
        Extracts and sums the log-probabilities of the actual token choices made by the model.
        
        Args:
            logits: Tensor of shape [batch_size, seq_len, vocab_size]
            labels: Tensor of shape [batch_size, seq_len] (padded with -100 for non-target tokens)
        """
        # Shift tokens by 1 to calculate autoregressive target transitions
        shift_logits = logits[..., :-1, :].contiguous()
        shift_labels = labels[..., 1:].contiguous()
        
        # Calculate cross entropy per token position without reduction
        per_token_logps = -F.cross_entropy(
            shift_logits.view(-1, shift_logits.size(-1)), 
            shift_labels.view(-1), 
            reduction='none'
        ).view(shift_labels.size())
        
        # Zero out padding values (where label is -100)
        padding_mask = (shift_labels == -100)
        per_token_logps[padding_mask] = 0.0
        
        # Sum along sequence dimension to get total log-probability for the generation
        return per_token_logps.sum(dim=-1)

    def forward(self, 
                policy_chosen_logits: torch.Tensor, policy_rejected_logits: torch.Tensor,
                reference_chosen_logits: torch.Tensor, reference_rejected_logits: torch.Tensor,
                chosen_labels: torch.Tensor, rejected_labels: torch.Tensor) -> tuple:
        
        # 1. Compute sequence log-probabilities under the active training policy
        policy_chosen_logps = self.get_batch_logps(policy_chosen_logits, chosen_labels)
        policy_rejected_logps = self.get_batch_logps(policy_rejected_logits, rejected_labels)
        
        # 2. Compute sequence log-probabilities under the frozen reference model
        with torch.no_grad():
            ref_chosen_logps = self.get_batch_logps(reference_chosen_logits, chosen_labels)
            ref_rejected_logps = self.get_batch_logps(reference_rejected_logits, rejected_labels)
            
        # 3. Calculate implicit log ratios
        chosen_log_ratios = policy_chosen_logps - ref_chosen_logps
        rejected_log_ratios = policy_rejected_logps - ref_rejected_logps
        
        # 4. Formulate the core DPO implicit reward subtraction term
        logits_diff = chosen_log_ratios - rejected_log_ratios
        
        # 5. Apply sigmoid and negative log likelihood calculation
        loss = -F.logsigmoid(self.beta * logits_diff).mean()
        
        # Track implicit metrics for monitoring optimization health
        with torch.no_grad():
            chosen_rewards = self.beta * chosen_log_ratios
            rejected_rewards = self.beta * rejected_log_ratios
            margins = chosen_rewards - rejected_rewards
            
        return loss, margins.mean().item()

# Verification execution pass
if __name__ == "__main__":
    B, T, V = 2, 8, 100 # Batch size=2, Sequence length=8, Vocab size=100
    dpo_criterion = DirectPreferenceOptimizationLoss(beta=0.1)
    
    # Simulate forward passes for Chosen (winning) and Rejected (losing) tokens
    mock_policy_chosen_logits = torch.randn(B, T, V, requires_grad=True)
    mock_policy_rejected_logits = torch.randn(B, T, V, requires_grad=True)
    
    mock_ref_chosen_logits = torch.randn(B, T, V)
    mock_ref_rejected_logits = torch.randn(B, T, V)
    
    # Mock token sequence indices matching inputs
    mock_chosen_labels = torch.randint(0, V, (B, T))
    mock_rejected_labels = torch.randint(0, V, (B, T))
    
    loss, reward_margin = dpo_criterion(
        mock_policy_chosen_logits, mock_policy_rejected_logits,
        mock_ref_chosen_logits, mock_ref_rejected_logits,
        mock_chosen_labels, mock_rejected_labels
    )
    
    print(f"Computed Analytical DPO Loss Vector: {loss.item():.4f}")
    print(f"Implicit Reward Margin (Chosen - Rejected): {reward_margin:.4f}")
```
